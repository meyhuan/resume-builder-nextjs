# 50 个简历模板规划

本文档用于规划 AI 简历构建器的模板库扩展方向，目标是在现有模板基础上扩展到 50 个模板，覆盖年轻中国求职用户的主流审美、岗位场景和投递需求。

## 目标用户

核心用户是年轻求职者，包括：

- 应届生、实习生、校招生
- 1-5 年经验的互联网、运营、产品、技术、设计、市场、新媒体用户
- 希望简历“精美、大气、有质感”的年轻白领
- 需要紧凑高效、一页承载更多信息的专业岗位用户
- 投递国企、银行、事业单位、传统行业的正式型用户

## 设计原则

1. 不做只是换颜色的模板。每个模板必须有明确用户、岗位和布局价值。
2. 年轻化不等于花哨。模板应高级、清爽、有记忆点，同时保持投递安全。
3. 模板库需要同时覆盖“精美大气”“紧凑专业”“校招活力”“正式稳重”“创意表达”。
4. 所有正式项目模板都必须保留现有编辑器能力，不得实现为静态 HTML。
5. 每个模板上线前必须按 `docs/template-creation-workflow.md` 执行 QA。

## 分类页与推荐标签

模板库建议支持以下筛选标签：

- 场景：校招、社招、实习、海投、国企、外企、管理岗
- 岗位：产品、运营、技术、设计、新媒体、财务、法务、行政、人事、销售、咨询
- 风格：简约、大气、精美、正式、创意、紧凑、双栏、表格、时间线
- 信息密度：低、中、高、很高
- 用户阶段：应届生、1-3 年、3-5 年、5 年以上、管理层

## 实现要求

每个新模板必须遵守项目模板工作流：

- 新增 `src/templates/<id>/index.tsx`
- 注册到 `src/templates/template-loader.ts`
- 注册到 `src/lib/templates/template-catalog.ts`
- 新增缩略图到 `public/thumbnails/`
- 如使用全出血页眉或特殊顶部布局，加入 `src/app/print/[id]/page.tsx` 的 `BLEED_TEMPLATE_IDS`
- 必须保留编辑器能力：
  - 基础信息编辑
  - 头像上传
  - 基础字段删除
  - 求职意向编辑
  - section 拖拽
  - section 删除
  - block 增删移动
  - AI 润色/生成
  - 富文本编辑
  - 移动端预览
  - 打印/导出
  - 主题控制

## QA 要求

每个模板实现后必须执行：

```bash
corepack pnpm template:qa <id> --local --base-url http://localhost:3000 --scenario-loader-url "http://localhost:3000/dev/scenario-loader?tpl=<id>" --report
corepack pnpm exec tsc --noEmit
corepack pnpm exec eslint <changed files>
```

如果 `localhost:3000` 不可用，按实际 dev server 端口改用 `127.0.0.1:<port>`。

## 产品策略建议

50 个模板不应一次性全部上线。建议按批次上线，每批 8-10 个，并观察：

- 模板点击率
- 选择率
- 编辑完成率
- 导出率
- 付费转化率
- 用户收藏/复用率

高表现模板可以扩展同风格变体，低表现模板应调整命名、缩略图、分类或视觉方向。
