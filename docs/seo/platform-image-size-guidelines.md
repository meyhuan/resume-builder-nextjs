# 平台图片尺寸规范

后续发布包要区分“源截图”和“平台成品图”。源截图可以是桌面截图或过程截图，真正发布前必须导出适合手机浏览的成品图。

## 总原则

- 封面优先，第一张图必须按平台推荐比例单独设计。
- 同一篇图文尽量统一比例，避免平台自动裁切或补边。
- 桌面截图不能直接铺满发布图，需要做竖版画布、标题、安全边距和局部放大。
- 产品 UI 必须清晰可读，手机上看不清的区域要做局部放大或裁切。

## 推荐尺寸

| 平台 | 封面 | 正文/图文 | 说明 |
| --- | --- | --- | --- |
| 小红书 | 1080x1440 或 1242x1660 | 1080x1440、1080x1350、1080x1080 | 竖版更适合信息流，同篇建议统一比例 |
| 公众号贴图 | 1080x1440 或 1242x1660 | 同封面比例统一 | 后续只维护贴图形态，首图承担封面点击 |
| 抖音图文 | 1242x1660、1080x1440 或 1080x1920 | 同封面比例统一 | 首图文字更短更大，主体放中间安全区 |
| 知乎 | 1200x675 或正文截图原比例 | 宽 900-1200 | 图片用于证明，不追求强封面 |

## 当前资产判断

- `1080x1350`：适合小红书 4:5 风格，也可作为抖音过渡图，但抖音更推荐 3:4 或 9:16。
- `1440x1000`：适合作为原始桌面截图，不适合直接作为小红书/抖音/公众号贴图首图。
- 公众号贴图使用 `1080x1440` 这类竖版成品图，不再按长文头图维护。

## 发布前检查

1. 第一张图是否是 `cover`。
2. 封面比例是否符合平台。
3. 手机宽度下主标题是否可读。
4. 产品核心区域是否没有被裁切。
5. 所有正文图比例是否统一。
6. 桌面截图是否做了局部放大或说明。
